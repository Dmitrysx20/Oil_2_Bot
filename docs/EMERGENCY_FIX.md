# 🚨 ЭКСТРЕННОЕ ИСПРАВЛЕНИЕ: Ошибка Docker сборки

## 🚨 ПРОБЛЕМА
```
ОШИБКА: недопустимая пара «ключ-значение» «= PERPLEXITY_API_KEY=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»: пустой ключ
```

## 🔍 ПРИЧИНА
В переменной окружения есть лишний символ `=` в начале, что делает ключ пустым.

## ✅ ЭКСТРЕННОЕ РЕШЕНИЕ

### Шаг 1: ПОЛНАЯ ОЧИСТКА в Railway Dashboard

1. **Откройте Railway Dashboard**
2. **Выберите ваш проект**
3. **Перейдите в раздел "Variables"**
4. **УДАЛИТЕ ВСЕ переменные окружения** (кнопка "Delete" для каждой)

### Шаг 2: Установка ТОЛЬКО базовых переменных

Добавьте **ТОЛЬКО** эти 4 переменные:

```
NODE_ENV=production
PORT=3000
NODE_NO_WARNINGS=1
NODE_OPTIONS=--no-deprecation --no-warnings
```

### Шаг 3: Перезапуск деплоя

1. **Нажмите "Redeploy"** в Railway Dashboard
2. **Дождитесь успешной сборки**
3. **Проверьте логи** - не должно быть ошибок

### Шаг 4: Добавление API ключей ПО ОДНОМУ

**ВАЖНО**: Добавляйте переменные по одной и проверяйте сборку после каждой!

1. **Добавьте первую переменную**:
   ```
   TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here
   ```
2. **Перезапустите деплой** и проверьте успешность
3. **Добавьте вторую переменную**:
   ```
   OPENAI_API_KEY=your_openai_api_key_here
   ```
4. **Повторите для остальных**:
   ```
   PERPLEXITY_API_KEY=your_perplexity_api_key_here
   SUPABASE_URL=your_supabase_url_here
   SUPABASE_ANON_KEY=your_supabase_anon_key_here
   ADMIN_CHAT_IDS=123456789,987654321
   ```

## 🔧 Автоматическое решение

Если у вас установлен Railway CLI:

```bash
# Установка CLI (если не установлен)
npm install -g @railway/cli

# Подключение к проекту
railway login
railway link

# Запуск скрипта диагностики
npm run railway:debug
```
