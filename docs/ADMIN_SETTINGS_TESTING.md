# Тестирование админских настроек

## Обзор

Система админских настроек включает в себя комплексную систему тестирования, которая проверяет все аспекты функциональности админских настроек бота ароматерапии.

## Тестовые скрипты

### 1. Основной тест админских настроек
```bash
node scripts/testAdminSettings.js
```

**Что тестирует:**
- Подключение к базе данных
- Инициализация админских настроек
- Получение и обновление настроек
- Обновление ежедневной статистики
- Статистика дашборда
- Специализированные методы
- Инкремент статистики
- Переключение настроек

### 2. Тест интеграции с админ-панелью
```bash
node scripts/testAdminSettingsIntegration.js
```

**Что тестирует:**
- Генерация админ-панели
- Обработка callback для настроек
- Переключение настроек через callback
- Состояние настроек после переключений
- Специализированные методы
- Статистика дашборда
- Обработка неизвестных callback
- Интеграция с другими админскими функциями

### 3. Тест миграций
```bash
node scripts/testAdminSettingsMigrations.js
```

**Что тестирует:**
- Проверка файлов миграций
- Скрипт запуска миграций
- Структура папки migrations
- Конфигурация
- Подключение к базе данных (если настроен Supabase)
- Синтаксис SQL файлов

### 4. Комплексный тест
```bash
node scripts/testAdminSettingsComprehensive.js
```

**Что тестирует:**
- Все аспекты админских настроек в одном скрипте
- Статистика прохождения тестов
- Подробная отчетность

## Структура тестирования

### Конфигурация
- Проверка настроек AdminService
- Проверка настроек AdminSettingsService
- Проверка переменных окружения

### Файлы миграций
- Существование всех необходимых файлов
- Содержимое миграции админских настроек
- Синтаксис SQL
- Баланс скобок

### Сервис админских настроек
- Инициализация настроек
- Получение настроек
- Обновление настроек бота
- Обновление ежедневной статистики

### Специализированные методы
- `isMaintenanceMode()` - режим обслуживания
- `shouldSendDailyStats()` - ежедневная статистика
- `isBroadcastEnabled()` - рассылки
- `shouldSendErrorNotifications()` - уведомления об ошибках
- `shouldSendNewUserNotifications()` - уведомления о новых пользователях

### Инкремент статистики
- `incrementErrors()` - инкремент ошибок
- `incrementNewUsers()` - инкремент новых пользователей
- `incrementActiveUsers()` - инкремент активных пользователей
- `incrementMessagesSent()` - инкремент отправленных сообщений

### Переключение настроек
- `setMaintenanceMode()` - переключение режима обслуживания
- `toggleDailyStats()` - переключение ежедневной статистики

### Админ-панель
- Генерация админ-панели
- Наличие кнопки настроек
- Обработка callback для настроек

### Переключатели
- `admin_maintenance_toggle`
- `admin_daily_stats_toggle`
- `admin_broadcast_toggle`
- `admin_error_notifications_toggle`
- `admin_new_user_notifications_toggle`

### Статистика дашборда
- Получение статистики дашборда
- Проверка всех необходимых полей

### Другие админские функции
- `admin_stats`
- `admin_users`
- `admin_broadcast`
- `admin_music`
- `admin_export`

### Обработка ошибок
- Обработка неизвестных callback
- Обработка некорректных параметров

## Режимы тестирования

### Mock режим
Когда Supabase не настроен, тесты работают в mock режиме:
- Все операции возвращают mock данные
- Функциональность проверяется без реальной базы данных
- Полезно для разработки и отладки

### Реальный режим
Когда Supabase настроен, тесты работают с реальной базой данных:
- Все операции выполняются в реальной базе данных
- Проверяется реальная функциональность
- Требует настройки переменных окружения

## Переменные окружения

Для полного тестирования требуются следующие переменные:

```bash
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_supabase_anon_key
ADMIN_CHAT_IDS=802895688,123456789
NODE_ENV=development
PORT=3000
```

### Статистика
- Общее количество тестов
- Количество пройденных тестов

## Примеры использования

### Запуск всех тестов
```bash
# Основной тест
node scripts/testAdminSettings.js

# Тест интеграции
node scripts/testAdminSettingsIntegration.js

# Тест миграций
node scripts/testAdminSettingsMigrations.js

# Комплексный тест
node scripts/testAdminSettingsComprehensive.js
```

### Проверка конкретной функциональности
```bash
# Только проверка файлов миграций
node scripts/testAdminSettingsMigrations.js

# Только интеграция с админ-панелью
node scripts/testAdminSettingsIntegration.js
```

## Устранение неполадок

### Ошибки подключения к базе данных
1. Проверьте настройки SUPABASE_URL и SUPABASE_ANON_KEY
2. Убедитесь, что миграции выполнены
3. Проверьте права доступа к таблице admin_settings

### Ошибки миграций
1. Проверьте существование всех файлов миграций
2. Убедитесь в корректности SQL синтаксиса
3. Проверьте баланс скобок в SQL файлах

### Ошибки интеграции
1. Проверьте настройки ADMIN_CHAT_IDS
2. Убедитесь в корректности callback обработчиков
3. Проверьте структуру админ-панели

## Логирование

Все тесты используют систему логирования:
- Информационные сообщения
- Предупреждения
- Ошибки
- Временные метки

Логи сохраняются в папке `logs/` для дальнейшего анализа.

## Автоматизация

Тесты можно интегрировать в CI/CD pipeline:
```bash
# В package.json
{
  "scripts": {
    "test:admin-settings": "node scripts/testAdminSettingsComprehensive.js",
    "test:admin-settings-migrations": "node scripts/testAdminSettingsMigrations.js"
  }
}
```

## Заключение

Система тестирования админских настроек обеспечивает:
- Полное покрытие функциональности
- Проверку интеграции с админ-панелью
- Валидацию миграций базы данных
- Обработку ошибок
- Подробную отчетность

Все тесты работают как в mock режиме (для разработки), так и в реальном режиме (для продакшена).
